- `Load()` takes no parameters — reads env vars via `os.Getenv`; tests control via `os.Setenv`/`os.Unsetenv`
- `Run(ctx, ln)` accepts context + listener — signal handling and listener creation are the caller's responsibility
- No build tags for Topic 1 — all tests self-contained, no K8s/NATS deps
- Test files use `_test` package suffix (e.g., `config_test`, `apiserver_test`) — Ginkgo convention
- `badRequestHandler` is a plain function (not a closure) — simple, no injected dependencies
- Request body validation enabled at the middleware layer — prepares for Topics 3/4
